# Export Feature: Implementation Plan

## Overview

The export feature reverses the populate workflow. It reads data from
PostgreSQL per data source, writes it into a fresh SFGA SQLite database
using `sfborg/sflib v0.5.6`'s `sfga.Archive` writer interface, and
packages the result using the same naming and output conventions as SFGA.

**Key insight:** `sfborg/sflib` already has a full writer interface
(`sfga.Archive`) with `InsertNames()`, `InsertTaxa()`, `InsertSynonyms()`,
`InsertVernaculars()`, `InsertMeta()`, and `Export()`. The SFGA writing
machinery already exists and is well-exercised by the `sf` tool.

**Reference:** Study `/home/dimus/code/golang/sf/pkg/from/` — especially
`shared.go` (InitSfga pattern) and `fcoldp/` (full Name+Taxon+Synonym
pipeline) for implementation patterns to follow.

## Output Files Per Source

For each exported data source, the following files are produced in the
output directory, all sharing the same base name
`{id:04d}-{slug}-{date}`:

| File | When produced |
|---|---|
| `0001-col-2025-08-25.sqlite` | always |
| `0001-col-2025-08-25.sql` | always |
| `0001-col-2025-08-25.sqlite.zip` | only with `--zip / -z` flag |
| `0001-col-2025-08-25.sql.zip` | only with `--zip / -z` flag |
| `0001-col-2025-08-25.yaml` | always — sources.yaml companion |

The date `2025-08-25` comes from `ds.RevisionDate` (the date embedded in
the original SFGA filename at import time), falling back to
`ds.UpdatedAt` if empty. See **File Naming Convention** below.

The `.sqlite` + `.sql` pair (and optional `.zip`) comes directly from
`arc.Export(outputPath, isZip)`. The `.yaml` is written separately by the
exporter.

### The companion `.yaml` file

The YAML file is a sources.yaml-compatible entry for the exported source.
Its purpose is to make the exported bundle re-importable: a user can copy
the YAML entry into their `sources.yaml` or `custom_sources.yaml` and run
`gndb populate` against the exported `.sqlite` file.

The `parent` field is set to `cfg.Export.ParentDir`, which defaults to
the output directory if `--parent` is not given. Since exported files are
typically uploaded to a server or shared location before being used, set
`--parent` to the final serving URL or path to produce a YAML that is
ready to use without editing.

**Example: `0001-col-2025-08-25.yaml`**

```yaml
# Generated by gndb export on 2025-08-25
# Add this entry to your sources.yaml or custom_sources.yaml to re-import.
id: 1
parent: /path/to/output/dir
title: "Catalogue of Life"
title_short: "Catalogue of Life"
home_url: "https://www.catalogueoflife.org/"
data_url: "https://www.catalogueoflife.org/data/download"
is_curated: true
has_classification: true
outlink_url: "https://www.catalogueoflife.org/data/taxon/{}"
outlink_id_column: "name.col__alternative_id"  # outlink IDs saved as gnoutlink: in AlternativeID
```

### Field destinations

Some fields go to **both** `coldp.Meta` inside the SFGA and the companion
YAML — making the SFGA self-contained while also supporting re-import.
Others are gndb-operational and belong only in the YAML.

| `schema.DataSource` | `coldp.Meta` field | YAML field |
|---|---|---|
| `Title` | `Title` | `title` |
| `TitleShort` | `Alias` | `title_short` |
| `Description` | `Description` | `description` (omit if empty) |
| `DOI` | `DOI` | — |
| `Citation` | `Citation` | — |
| `WebsiteURL` | `URL` | `home_url` |
| `Version` | `Version` | — |
| `RevisionDate` | `Issued` | — |

| `schema.DataSource` | YAML-only field | Notes |
|---|---|---|
| `ID` | `id` | gndb source identifier |
| `cfg.Export.ParentDir` | `parent` | `--parent` flag; defaults to output dir |
| `DataURL` | `data_url` | omit if empty |
| `IsCurated` | `is_curated` | omit if false |
| `IsAutoCurated` | `is_auto_curated` | omit if false |
| `HasTaxonData` | `has_classification` | best proxy available |
| auto (see outlink section) | `outlink_id_column` | omit if no outlink |

### Outlink ID preservation

`name_string_indices.outlink_id` holds the actual outlink ID value for
each record. During export, this is written into `coldp.Name.AlternativeID`
with the scope prefix `gnoutlink:`:

```
Name.AlternativeID = "gnoutlink:" + nsi.OutlinkID   // when OutlinkID != ""
```

This follows the CoLDP convention for `AlternativeID`:
`scope:id` (e.g. `col:123`, `gbif:uuid`).

Because the outlink ID is now consistently stored in
`name.col__alternative_id`, the companion YAML can auto-populate
`outlink_id_column` whenever the source has outlink data:

```yaml
outlink_url: "https://www.catalogueoflife.org/data/taxon/{}"
outlink_id_column: "name.col__alternative_id"   # auto-set, no longer manual
```

**Query note:** `outlink_id` lives on `name_string_indices` (record
level), but `coldp.Name` is per name-string. A single name-string can
appear in multiple `name_string_indices` rows with different
`outlink_id` values. Take the first non-empty `outlink_id` from the
JOIN — in practice, outlink IDs are consistent per name-string within
a source.

### Fields NOT in PostgreSQL (lost after populate)

| `sources.yaml` field | Status |
|---|---|
| `outlink_id_column` | ✅ auto-set to `"name.col__alternative_id"` when `outlink_url` is present |
| `prefer_flat_classification` | not written — irrelevant for exported data |

The exported SFGA already contains the classification as it was stored in
PostgreSQL. `prefer_flat_classification` was an import-time hint for
choosing between two classification styles present in the source SFGA; it
has no meaning when re-importing an export from gnames.

## CLI Command

**`cmd/export.go`** — mirrors `cmd/populate.go`:

```
gndb export [--source-ids 1,2,3] [--output-dir /path/to/output] [--zip]
```

Flags:
- `--source-ids / -s []int` — data source IDs to export (empty = all)
- `--output-dir / -o string` — destination directory (default: current dir)
- `--parent / -p string` — value written to `parent` field in companion
  YAML (default: output directory path). Set this to the URL or path from
  which the exported files will actually be served, e.g.
  `http://myserver.org/sfga/` or `/mnt/shared/sfga/`.
- `--zip / -z bool` — also produce `.zip` compressed variants (default:
  false)

The `runExport()` function flow:
1. Build export options from changed flags, apply to `cfg`
2. Create `iodb.NewPgxOperator()`, connect
3. Check `op.HasTables(ctx)` — must have tables
4. Create `ioexport.New(cfg, op)`
5. Call `exporter.Export()`

## Orchestrator: `internal/ioexport/exporter.go`

```go
type exporter struct {
    cfg      *config.Config
    operator db.Operator
}

func New(cfg *config.Config, op db.Operator) gndb.Exporter {
    return &exporter{cfg: cfg, operator: op}
}
```

`Export()` top-level flow:
1. Query `data_sources` table, filter by `cfg.Export.SourceIDs` (empty = all)
2. For each source call `exportSource(ds schema.DataSource)`
3. Track success/error counts, report totals (same pattern as
   `processSources` in `iopopulate`)

`exportSource(ds)` stages:
1. Validate output directory exists (create if needed)
2. Create temp work directory under `~/.cache/gndb/export/{source_id}/`
3. Call `InitSfga(workDir)` → create + connect SFGA SQLite
4. **(1/5) metadata** — build `coldp.Meta` from `ds`, call
   `arc.InsertMeta()`
5. **(2/5) names** — batch-read `name_strings` for this source, call
   `arc.InsertNames()`
6. **(3/5) taxa** — batch-read accepted `name_string_indices`, call
   `arc.InsertTaxa()`
7. **(4/5) synonyms** — batch-read synonym `name_string_indices`, call
   `arc.InsertSynonyms()`
8. **(5/5) vernaculars** — batch-read `vernacular_string_indices` + join,
   call `arc.InsertVernaculars()`
9. Build output base path: `{outputDir}/{0001-col-2025-08-25}`
10. Call `arc.Export(basePath, cfg.Export.WithZip)` → produces `.sqlite`
    + `.sql` (and `.zip` variants if `WithZip`)
11. Write `{basePath}.yaml` — sources.yaml-compatible companion file
12. Deferred cleanup of temp work directory

## `internal/ioexport/sfga.go` — SFGA Helpers

Mirrors the `from.Shared.InitSfga()` pattern from the `sf` project:

```go
func initSfga(workDir string) (sfga.Archive, error) {
    slog.Info("Creating SFGA database")
    arc := sflib.NewSfga()
    if err := arc.Create(workDir); err != nil {
        return nil, err
    }
    if _, err := arc.Connect(); err != nil {
        return nil, err
    }
    return arc, nil
}
```

Also contains:
- `buildOutputPath(ds schema.DataSource, outputDir string) string`
- `slugify(s string) string`
- `ensureOutputDir(dir string) error`
- `writeCompanionYAML(ds schema.DataSource, basePath string) error`

**`writeCompanionYAML`** marshals a `sources.DataSourceConfig` populated
from `schema.DataSource` fields to `{basePath}.yaml`. Sets `parent` to
`cfg.Export.ParentDir` (falling back to `cfg.Export.OutputDir` if empty).
Includes commented-out placeholders for the two fields not available from
PostgreSQL (`outlink_id_column`, `prefer_flat_classification`). Uses
`gopkg.in/yaml.v3` (already a transitive dependency).

**Note:** `arc.Create()` fetches the SFGA schema SQL from a remote Git
repo. Requires internet access. Wrap with clear error if fetch fails.

## Data Queries

### Sources (at startup)

```sql
SELECT id, title, title_short, version, revision_date, doi, citation,
       authors, description, website_url, data_url, outlink_url,
       is_curated, is_auto_curated, has_taxon_data,
       record_count, vern_record_count
FROM data_sources
ORDER BY id
```

Filter to `cfg.Export.SourceIDs` in Go after loading if non-empty.

### Names (per source, paginated)

`name_strings` is global — filter via JOIN through `name_string_indices`.
Also pick up the first non-empty `outlink_id` per name-string for
`AlternativeID`:

```sql
SELECT DISTINCT ON (ns.id)
    ns.id, ns.name, ns.year, ns.cardinality,
    ns.virus, ns.bacteria, ns.surrogate, ns.parse_quality,
    c.name  AS canonical_simple,
    cf.name AS canonical_full,
    nsi.outlink_id
FROM name_strings ns
JOIN name_string_indices nsi ON nsi.name_string_id = ns.id
LEFT JOIN canonical c        ON c.id  = ns.canonical_id
LEFT JOIN canonical_full cf  ON cf.id = ns.canonical_full_id
WHERE nsi.data_source_id = $1
ORDER BY ns.id, nsi.outlink_id NULLS LAST
LIMIT $2 OFFSET $3
```

`DISTINCT ON (ns.id) ... ORDER BY ns.id, nsi.outlink_id NULLS LAST`
picks one row per name-string, preferring rows that have a non-null
`outlink_id`.

### Taxa (accepted, per source, paginated)

```sql
SELECT
    record_id, name_string_id, outlink_id,
    global_id, name_id, local_id, code_id,
    rank, taxonomic_status, accepted_record_id,
    classification, classification_ids, classification_ranks
FROM name_string_indices
WHERE data_source_id = $1
  AND (taxonomic_status NOT IN ('synonym', 'ambiguous synonym', 'misapplied')
       OR taxonomic_status IS NULL)
ORDER BY record_id
LIMIT $2 OFFSET $3
```

### Synonyms (per source, paginated)

```sql
SELECT
    record_id, accepted_record_id, name_string_id,
    taxonomic_status, code_id
FROM name_string_indices
WHERE data_source_id = $1
  AND taxonomic_status IN ('synonym', 'ambiguous synonym', 'misapplied')
ORDER BY record_id
LIMIT $2 OFFSET $3
```

### Vernaculars (per source, paginated)

```sql
SELECT
    vsi.record_id, vs.name,
    vsi.language_orig, vsi.language, vsi.lang_code,
    vsi.locality, vsi.country_code, vsi.preferred
FROM vernacular_string_indices vsi
JOIN vernacular_strings vs ON vs.id = vsi.vernacular_string_id
WHERE vsi.data_source_id = $1
ORDER BY vsi.record_id
LIMIT $2 OFFSET $3
```

## Classification Reconstruction

PostgreSQL stores classification as pipe-delimited strings built by
`getBreadcrumbs()` during populate:

```
classification:       "Animalia|Chordata|Mammalia|Carnivora|Felidae|Felis"
classification_ranks: "kingdom|phylum|class|order|family|genus"
classification_ids:   "1|5|6|7|8|9"
```

**`classification_ranks` is always populated** when classification exists
— both hierarchical and flat classification paths always store ranks.
**`classification_ids` may be empty** when the source only had flat
classification without IDs.

### Case 1: ranks + IDs present (hierarchical source)

Both flat fields and `ParentID` can be populated. The IDs in
`classification_ids` are real `record_id` values from this source's
`name_string_indices`. Set `ParentID` to the penultimate element.

```go
func applyClassification(t *coldp.Taxon, clsf, ranks, ids string) {
    names := strings.Split(clsf, "|")
    rnks  := strings.Split(ranks, "|")
    idArr := strings.Split(ids, "|")

    for i, rank := range rnks {
        if i >= len(names) {
            break
        }
        id := ""
        if i < len(idArr) {
            id = idArr[i]
        }
        switch rank {
        case "kingdom":   t.Kingdom = names[i];   t.KingdomID = id
        case "phylum":    t.Phylum  = names[i];   t.PhylumID  = id
        case "subphylum": t.Subphylum = names[i]; t.SubphylumID = id
        case "class":     t.Class   = names[i];   t.ClassID   = id
        case "subclass":  t.Subclass = names[i];  t.SubclassID = id
        case "order":     t.Order   = names[i];   t.OrderID   = id
        case "suborder":  t.Suborder = names[i];  t.SuborderID = id
        case "superfamily": t.Superfamily = names[i]; t.SuperfamilyID = id
        case "family":    t.Family  = names[i];   t.FamilyID  = id
        case "subfamily": t.Subfamily = names[i]; t.SubfamilyID = id
        case "tribe":     t.Tribe   = names[i];   t.TribeID   = id
        case "subtribe":  t.Subtribe = names[i];  t.SubtribeID = id
        case "genus":     t.Genus   = names[i];   t.GenusID   = id
        case "subgenus":  t.Subgenus = names[i];  t.SubgenusID = id
        case "species":   t.Species = names[i];   t.SpeciesID = id
        }
    }

    // ParentID = second-to-last element of classification_ids
    // (valid only when IDs are real record_ids from this source)
    if len(idArr) >= 2 && idArr[len(idArr)-2] != "" {
        t.ParentID = idArr[len(idArr)-2]
    }
}
```

### Case 2: ranks present, IDs empty (flat-only source)

Populate flat fields exactly as above, but skip `ParentID` — there are
no real record IDs to reference.

**Possible future enhancement: unflattening.** When we have names + ranks
but no IDs, we could synthesize virtual parent taxon records from the
unique classification paths (e.g. all taxa sharing `Kingdom=Animalia,
Phylum=Chordata` get a synthetic `Chordata` node as parent). The `sf`
tool already implements this via `--add-parents`. Whether to attempt this
during export is deferred — it is non-trivial and requires deduplication
across the entire source.

### Case 3: ranks empty (with or without IDs)

`classification_ranks` is empty but `classification` names are present.
Rather than dropping the entire breadcrumb, attempt rank inference using
two complementary strategies:

- **Strategy A — Suffix rules** (family and below): ICN/ICZN mandate
  specific suffixes for formal ranks; `-inae` ambiguity is resolved via
  `nsi.CodeID` or a breadcrumb-wide majority-code vote.
- **Strategy B — Higher-taxon dictionary** (order and above): a YAML
  file embedded at build time (`internal/ioexport/data/rankdict.yaml`)
  maps ~1,500 Animalia and Plantae taxa to rank + nomenclatural code.
  Generated from the gnames database via `gndb gen-rankdict`.

If a name's rank cannot be inferred by either strategy, that node is
silently skipped — a partial breadcrumb is better than none.

See **`UNRANKED_BREADCRUMBS_PLAN.md`** for the full inference pipeline,
suffix table, dictionary structure, SQL query, and `gen-rankdict` command
design.

## File Naming Convention

SFGA canonical format: `{ID:04d}-{title_short_slug}-{date}`

### Date selection

Two date fields exist in `schema.DataSource`:

| Field | Origin | Meaning |
|---|---|---|
| `RevisionDate string` | SFGA filename at populate time | When source data was **published** |
| `UpdatedAt time.Time` | `time.Now()` at populate time | When data was **imported** into PostgreSQL |

Use `RevisionDate` as the primary date — it came from the original SFGA
filename and identifies the data version. Fall back to `UpdatedAt` if
`RevisionDate` is empty (it is always set, so this is a reliable fallback).

```go
date := ds.RevisionDate
if date == "" {
    date = ds.UpdatedAt.Format("2006-01-02")
}
```

Do **not** use `time.Now()` — the export date has no relation to the data
itself and would produce different filenames on each export run.

### Other naming rules

- ID zero-padded to 4 digits: `0001`, `0011`, `0132`, `1000`
- `title_short_slug`: lowercase `TitleShort`, spaces → `-`,
  non-alphanumeric characters stripped
- If `TitleShort` is empty, fall back to `"source"`

### Examples

| `schema.DataSource` | Base filename |
|---|---|
| `{1, "Catalogue of Life", "CoL", "2025-08-25", ...}` | `0001-col-2025-08-25` |
| `{132, "ITIS", "ITIS", "2024-12-01", ...}` | `0132-itis-2024-12-01` |
| `{1000, "My Source", "My Source", "", UpdatedAt: 2026-03-23}` | `1000-my-source-2026-03-23` |

## Batch Reading Strategy

Use `OFFSET/LIMIT` pagination with `cfg.Database.BatchSize` (default
50,000). For each stage, a simple loop:

```go
for offset := 0; ; offset += batchSize {
    batch, err := queryNamesBatch(ctx, pool, sourceID, batchSize, offset)
    if err != nil {
        return err
    }
    if len(batch) == 0 {
        break
    }
    if err = arc.InsertNames(batch); err != nil {
        return err
    }
    // update progress bar
}
```

**Large-source optimization:** For the names query, the `DISTINCT JOIN`
may be slow for sources with millions of records (CoL ~10M). Alternative:
collect all `name_string_id` UUIDs from `name_string_indices` first (one
pass), then batch-fetch `name_strings` by those IDs to avoid repeated
full-table scans.

## Progress Reporting

Follow the same pattern as `iopopulate`:
- Per-phase `gn.Info("(N/5) ...")` and `gn.Message(...)` with elapsed time
- `cheggaaa/pb/v3` progress bars for long operations (names, taxa)
- `gnfmt.TimeString()` for elapsed time

## Error Handling

- Per-source: log error, increment `errorCount`, continue with next source
  (same as `processSources`)
- If all sources fail: return `AllSourcesFailedError`
- Deferred cleanup of temp work directory on error
- Clear error for `arc.Create()` network failure (schema fetch)

## Network Dependency: SFGA Schema Fetch

`sflib.Archive.Create()` fetches the SFGA schema SQL from a remote Git
repo. Requires internet access during export. Wrap with a clear error
message if this fails, e.g.:

```
Failed to create SFGA archive for source 1.
This step requires internet access to fetch the SFGA schema.
Check your connection and try again.
```

**No `sqlite3` CLI dependency** — `arch.Packager.Export()` handles the
output entirely within Go; no shell-out to `sqlite3` required (confirmed
from `arch/interface.go`).

## Known Challenges

1. **`name_strings` is a global table** — filter requires JOIN through
   `name_string_indices`. For large sources, collect `name_string_id`
   values first, then batch-fetch by IDs.

2. **`coldp.Taxon` has no `Rank` field** — rank lives on `coldp.Name`,
   not `coldp.Taxon`. The `rank` from `name_string_indices` should be set
   on the `Name` record, not the `Taxon`. (To verify: check if there is
   actually no rank on Taxon in the struct — the struct read above confirms
   this: `Taxon` has no `Rank` field. Use `coldp.NewRank()` on the `Name`
   struct.)

3. **Authorship split** — PostgreSQL does not store authorship separately.
   Set `Name.ScientificName = canonical_simple`, leave `Name.Authorship`
   empty; `ScientificNameString` carries the full string.

4. **Taxon flat classification ranks** — PostgreSQL stores only
   kingdom/phylum/class/order/family/genus (no subtribe, tribe, subfamily,
   suborder, subclass, subphylum, subgenus, etc.) in the classification
   breadcrumb. Only map what is present; leave others empty.

5. **`classification_ids` may not match taxon `record_id`** — the IDs in
   the breadcrumb are `record_id` values from the same data source. They
   should be usable as `ParentID` directly, but this needs to be confirmed
   against real data before relying on it.
