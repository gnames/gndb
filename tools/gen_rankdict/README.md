# gen_rankdict

Developer maintenance tool that generates `internal/ioexport/data/rankdict.yaml`
— a dictionary of higher taxa (kingdom through suborder) for Animalia and
Plantae, each annotated with rank and nomenclatural code (iczn/icn).

The dictionary is used by `gndb export` to infer ranks for classification
breadcrumb nodes that were stored without rank information. 

## Prerequisites

A populated gnames PostgreSQL database with at least one of:

- Data source 1 — Catalogue of Life
- Data source 3 — ITIS
- Data source 9 — WoRMS

Connection settings are read from `GNDB_DATABASE_*` environment variables.
The easiest way to set these is via the project's `.envrc` (direnv):

```sh
cp .envrc.example .envrc
# edit .envrc with your database credentials
direnv allow .
```

## Usage

Run from the repository root:

```sh
go run tools/gen_rankdict/main.go
```

The output is written to `internal/ioexport/data/rankdict.yaml` by default.
Override with `--output`:

```sh
go run tools/gen_rankdict/main.go --output /tmp/rankdict.yaml
```

## When to regenerate

Regenerate after importing a new version of CoL or ITIS, then commit the
updated file:

```sh
gndb populate -s 1   # re-import Catalogue of Life
go run tools/gen_rankdict/main.go
git add internal/ioexport/data/rankdict.yaml
git commit -m "update rankdict.yaml from CoL YYYY-MM-DD"
```

## Output format

YAML grouped by rank, alphabetically sorted within each group, columns
aligned for readability:

```yaml
# Generated by tools/gen_rankdict.go on 2025-08-25
# Sources: CoL (1), ITIS (3), WoRMS (9)
# Scope: Animalia (iczn) + Plantae (icn) only
# Total entries: 1104
---

# kingdoms (2)
Animalia: {rank: kingdom, code: iczn}
Plantae:  {rank: kingdom, code: icn}

# phyla (63)
Acanthocephala: {rank: phylum, code: iczn}
Annelida:       {rank: phylum, code: iczn}
...

# orders (831)
Accipitriformes: {rank: order, code: iczn}
Agaricales:      {rank: order, code: icn}
...
```

Source priority when the same name appears in multiple sources: CoL > ITIS > WoRMS.
Entries with an unresolvable code (`unknown`) are silently dropped.
